# 开发指南

## 环境要求

- **Node.js** >= 20
- **Bun** >= 1.3 (API 运行环境)
- **PostgreSQL** >= 15
- **pnpm** >= 10

## 快速开始

```bash
# 克隆项目
git clone <repo-url> && cd super-app

# 安装依赖
pnpm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入必要配置

# 启动开发环境（所有应用 + API + Worker 一并启动）
pnpm dev
```

## .env 关键配置

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | PostgreSQL 连接串 |
| `DASHSCOPE_API_KEY` | 阿里云百炼 API Key |
| `STORAGE_DRIVER` | `local` 或 `oss` |
| `SESSION_SECRET` | 会话加密密钥（至少 32 字符） |

## 常用命令

```bash
pnpm dev          # 启动所有服务
pnpm build        # 构建所有包和应用
pnpm typecheck    # 类型检查
pnpm lint         # 代码规范检查
pnpm test         # 运行测试

# 单个包
pnpm --filter @super-app/api dev
pnpm --filter @super-app/api test

# 数据库
pnpm db:generate  # 生成迁移
pnpm db:migrate   # 执行迁移
pnpm db:studio    # Drizzle Studio
```

## API 开发

API 模块位于 `services/api/src/modules/<name>/`：

```
index.ts   # Elysia 路由
service.ts # 业务逻辑
```

新增模块后需在 `services/api/src/app.ts` 注册路由。

## Worker 开发

Worker handler 位于 `services/worker/src/handlers/`：

```typescript
export const myHandler: TaskHandler = async (task) => {
  // 1. 执行任务
  // 2. 更新 generation_record 状态
  // 3. 调用 billing 结算
  // 4. 发送 SSE 通知
}
```

注册到 `task-handlers.ts` 的 `definitions` 数组。

## 数据库

使用 Drizzle ORM。Schema 定义在 `packages/db/src/schema/`。
Repository 层在 `packages/db/src/repositories/`。

```bash
# 修改 schema 后
pnpm db:generate   # 生成 SQL 迁移
pnpm db:migrate    # 执行迁移
```
